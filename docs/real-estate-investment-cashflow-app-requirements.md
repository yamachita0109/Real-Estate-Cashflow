# 不動産投資キャッシュフロー計算アプリ要件

## 目的

不動産投資物件の初期費用、月次キャッシュフロー、年次キャッシュフロー、退去費用を一元計算し、投資判断に必要な収支構造とリスクをグラフィカルに可視化するアプリケーション。

## 対象ユーザー

- 個人不動産投資家
- 物件購入前に収支を確認したい投資初心者から中級者
- 区分マンション、一棟アパート、一棟マンションの購入検討者
- 金融機関提出用ではなく、購入判断用のシミュレーション利用者

## コア価値

- 自己資金の必要額把握
- 月次黒字化の可否判定
- 年間収支と利回りの把握
- 退去発生時の損失インパクト把握
- 金利、空室率、家賃下落への耐性確認

## MVP範囲

- 単一物件の収支シミュレーション
- 税引前キャッシュフロー計算
- 元利均等返済によるローン計算
- 初期費用、月次CF、年次CF、退去費用の自動計算
- 入力変更に対するリアルタイム再計算
- グラフによる収支可視化
- ブラウザ内ローカル保存
- CSVエクスポート
- PC、スマートフォン対応
- Vercelへのデプロイ

## MVP対象外

- 所得税、住民税、法人税の詳細計算
- 減価償却計算
- 売却益、譲渡税計算
- 複数物件ポートフォリオ管理
- ログイン、ユーザー管理
- クラウド保存
- 金融機関提出用帳票
- 外部不動産サイト連携

## 計算仕様

### 初期費用

```text
初期費用合計 = 頭金
  + 仲介手数料
  + 登記費用
  + 司法書士報酬
  + 印紙税
  + 不動産取得税
  + 固定資産税等精算金
  + 保険料初年度分
  + 融資関連費用
  + 初期リフォーム費用
  + その他購入時費用
```

```text
必要自己資金 = 初期費用合計 - 融資に組み込む諸費用
```

### 月次収入

```text
月次満室収入 = 月額家賃 + 共益費収入 + 駐車場収入 + その他収入
```

```text
実効月次収入 = 月次満室収入 * (1 - 空室率)
```

### 月次運営費

```text
月次運営費 = 管理委託費
  + 建物管理費
  + 修繕積立金
  + 固定資産税月割
  + 都市計画税月割
  + 保険料月割
  + 修繕予備費
  + その他月次費用
  + その他年次費用月割
```

### ローン返済

```text
月利 = 年利 / 12
返済回数 = 借入期間年数 * 12
月額返済 = 融資額 * 月利 * (1 + 月利)^返済回数 / ((1 + 月利)^返済回数 - 1)
```

金利0%の場合。

```text
月額返済 = 融資額 / 返済回数
```

### 月次キャッシュフロー

```text
NOI = 実効月次収入 - 月次運営費
月次CF = NOI - 月額ローン返済
```

### 年次キャッシュフロー

```text
年間満室収入 = 月次満室収入 * 12
年間実効収入 = 実効月次収入 * 12
年間運営費 = 月次運営費 * 12
年間NOI = NOI * 12
年間ローン返済 = 月額ローン返済 * 12
年次CF = 月次CF * 12
```

### 利回り指標

```text
表面利回り = 年間満室収入 / 物件価格
実質利回り = 年間NOI / 物件価格
自己資金利回り = 年次CF / 必要自己資金
DSCR = 年間NOI / 年間ローン返済
```

### 退去費用

```text
空室損失 = 月次満室収入 * 空室期間月数
退去一時費用 = 原状回復費
  + クリーニング費
  + 募集広告費
  + 客付け仲介手数料
  + 鍵交換費用
  - 敷金精算額
```

```text
退去イベント損失 = 空室損失 + 退去一時費用
年換算退去コスト = 退去イベント損失 / 想定入居期間年数
退去考慮後年次CF = 年次CF - 年換算退去コスト
```

### 損益分岐

```text
損益分岐月額家賃 = 月次運営費 + 月額ローン返済
損益分岐空室率 = 1 - ((月次運営費 + 月額ローン返済) / 月次満室収入)
```

## 投資判定ロジック

### 良好

- 月次CFがプラス
- DSCRが1.2以上
- 自己資金利回りが目標値以上
- 退去考慮後年次CFがプラス
- 損益分岐空室率が20%以上

### 注意

- 月次CFが小幅プラスまたは小幅マイナス
- DSCRが1.0以上1.2未満
- 退去考慮後年次CFが小幅マイナス
- 金利上昇または空室率上昇で赤字化

### 危険

- 月次CFが明確なマイナス
- DSCRが1.0未満
- 退去考慮後年次CFが大幅マイナス
- 損益分岐空室率が10%未満

## UX要件

- 入力変更時の即時再計算
- 数値入力の単位表示
- 円、万円、パーセント、年、月の明確な区別
- 異常値入力時の警告表示
- 計算不能時の理由表示
- スマートフォンでの片手操作に耐えるフォーム幅
- PCでの入力と結果の並列表示
- グラフと数値の同時確認

## データ保存要件

- 既定の保存先はブラウザローカルストレージ
- Cookie保存はローカルストレージが利用できない場合のフォールバック、またはユーザーが明示選択した場合のみ利用
- 最終入力値の自動復元
- 入力変更時の自動保存
- 保存対象は物件情報、融資条件、収入条件、運営費条件、初期費用条件、退去条件、表示設定
- 保存キーはアプリケーション専用の名前空間を利用
- Cookie利用時は必要最小限のデータのみ保存
- Cookie利用時はSameSite=Laxを指定
- Cookie利用時は保存期限を明示
- 保存データの削除操作
- 保存データの破損時は初期値へ復旧
- サンプルシナリオの読み込み
- CSVエクスポート
- 将来的なJSONインポート、エクスポート余地

## 技術要件

- Vercelでホスティング可能なフロントエンドアプリケーション
- ReactまたはNext.jsによる実装
- MVPではバックエンドAPIなし
- 計算処理はブラウザ内で完結
- 保存処理はブラウザローカルストレージまたはCookieで完結
- グラフ描画ライブラリの利用
- TypeScriptによる計算ロジックの型安全性確保
- 計算ロジックとUIコンポーネントの分離
- 静的配信可能なビルド成果物

## Vercelデプロイ要件

- GitHub連携による自動デプロイ
- mainブランチへのマージ時のProductionデプロイ
- Pull Request作成時のPreviewデプロイ
- ビルドコマンドの明示
- 出力ディレクトリの明示
- 環境変数なしで動作するMVP
- サーバーサイドセッション非利用
- Vercel Edge Middleware非利用
- SPAルーティング利用時のリロード対応
- デプロイ後URLでのPC、スマートフォン表示確認

## 非機能要件

- 初回表示3秒以内
- 入力変更後の再計算100ms以内
- スマートフォン幅375px以上への対応
- デスクトップ幅1440px前後での視認性確保
- オフライン動作
- 税務上の正確性を保証しない旨の明記

## 受け入れ条件

- 初期費用、月次CF、年次CF、退去費用が個別に確認可能
- 入力変更時にサマリーとグラフが同期更新
- 月額ローン返済額が融資条件から自動算出
- 退去イベント損失と退去考慮後年次CFの表示
- 表面利回り、実質利回り、自己資金利回り、DSCRの表示
- 空室率、金利、家賃下落率による感度分析の表示
- PCとスマートフォンでレイアウト崩れなし
- CSVとして計算結果を出力可能
